# 留痕 Trace：公開測試版產品與開發規劃書

> 文件版本：1.1
>
> 文件日期：2026-07-15
>
> 產品階段：可編譯探索版 → TestFlight 外部測試
>
> 初期平台：iPhone／iOS 17 以上
>
> 招募通路：Threads、專業人士與勞權社群轉介

## 1. 規劃結論

目前 Repo 已具有可編譯的 SwiftUI App、SwiftData、本機附件保存、OCR、Share Extension、Face ID、加密備份、單事件 PDF、視覺設計與 App Icon。

下一個目標不是增加更多展示功能，而是完成一個能安全交給真實使用者的外部測試版：

> 使用者能主動匯入一組聊天截圖，保留不可覆寫的原始檔，確認 OCR、人物、順序與事件關聯，最後輸出可交給專業人士理解的案件包；整個過程預設不離開裝置。

TestFlight 公開前必須補齊資料不可變、完整刪除、對話整理、案件包、真機安全測試、語音轉錄、Apple Intelligence 整理與隱私文件。StoreKit 付費功能不列為首輪公開測試阻擋條件。

語音與 Apple Intelligence 採能力分級，不提高整個 App 的 iOS 17 最低版本：

- 所有支援裝置都能主動錄音、匯入既有音檔、保存原始檔及手動整理。
- iOS 26 以上使用 `SpeechAnalyzer／SpeechTranscriber` 產生裝置端逐字稿。
- 支援 Apple Intelligence 的裝置使用 Foundation Models 產生結構化事件草稿。
- 不支援的裝置顯示原因並提供固定問答流程，不將事件內容自動上傳第三方 AI。

## 2. 目前基準

### 已完成

- SwiftUI App 與 SwiftData 資料模型。
- 建立、編輯與刪除事件的基本畫面。
- 從 Photos、Files 匯入圖片、PDF 與音檔。
- Share Extension 接收多張圖片並透過 App Group 暫存。
- 原始附件保存、匯入時間與 SHA-256 計算。
- Vision 本機 OCR 與使用者確認欄位。
- 單事件摘要 PDF。
- Face ID／Touch ID／裝置密碼畫面保護。
- 密碼加密備份及還原的第一版。
- 「溫和留痕／案件編目／私密保險箱」視覺系統。
- 1024×1024 App Icon。
- GitHub Actions macOS Simulator 編譯驗證。

### 尚未達到公開測試標準

- 原始陳述與原始 OCR 仍可從 UI 直接修改。
- 缺少資料修訂版本與稽核紀錄。
- 刪除事件時，附件實體檔可能留在裝置中。
- Share Extension 的多圖完成順序可能與使用者選取順序不同。
- 沒有聊天截圖專用的人物、訊息順序及跨圖分組流程。
- PDF 仍是單事件摘要，不是多事件案件包。
- 缺少完整原始附件 ZIP 與 manifest。
- App 切換器隱私遮罩尚未完成。
- 尚未實作 App 內錄音、SpeechAnalyzer 逐字稿與逐段確認。
- 尚未實作 Foundation Models 結構化草稿、可用性檢查及手動降級。
- 備份、刪除、OCR 與匯出的自動化測試尚未建立。
- 尚未完成 Apple Developer App Group、簽署與真機驗證。
- 尚無隱私權政策、測試者同意與 App 內回饋入口。

## 3. 公開測試目標

### 使用者目標

使用者可在不理解法律術語的情況下完成：

```text
保存原始截圖
→ 確認對話文字、人物、日期與順序
或：主動錄音／匯入音檔
→ 確認逐字稿
→ 由 Apple Intelligence 協助整理並由使用者確認
→ 關聯一件或多件工作事件
→ 預覽事件時間線
→ 匯出 PDF／完整附件包
```

### 產品驗證目標

首輪 TestFlight 只驗證以下問題：

1. 使用者是否願意把真實但可匿名化的聊天截圖加入留痕。
2. OCR 確認、人物對應與事件關聯是否比 Word／相簿整理省力。
3. 專業人士是否能更快理解事件脈絡與缺漏。
4. 使用者是否信任資料預設保留在裝置上。
5. 完整案件包是否具有可付費價值。
6. 語音轉錄是否能降低會議或事後回想的記錄負擔。
7. Apple Intelligence 草稿是否能縮短整理事件與找出資訊缺口的時間。

## 4. 不可妥協原則

1. 原始附件、原始陳述與系統原始 OCR 不可覆寫。
2. 所有修正以新版本保存，並能追溯來源。
3. SHA-256 只驗證匯入後完整性，不宣稱證明真偽。
4. AI 只產生待確認草稿，不作法律結論。
5. App 不讀取 LINE 帳號、聊天資料庫或通知內容。
6. 不提供隱蔽錄音或背景持續錄音。
7. 使用者控制備份、匯出、分享與刪除。
8. 不因加入 AI 而讓舊裝置無法完成核心流程。
9. 錄音必須由使用者明確開始，畫面持續顯示錄音狀態，不自動啟動。
10. 只有使用者已確認的逐字稿或 OCR 文字可進入 AI 整理；原始音檔與圖片不直接交給語言模型。

## 5. 開發階段與 Gate

本規劃採 Gate 制，不以固定週數自動前進。每階段通過驗收才進入下一階段。

### M0：資料完整性與安全刪除

#### 交付內容

- 建立 `EventRevision`，原始陳述唯讀，修正另存版本。
- 建立 `OCRResult` 與 `ConfirmedTranscript` 分層。
- 新增附件完整性重新驗證。
- 刪除單一附件時移除資料庫紀錄、原始檔及衍生檔。
- 刪除事件時遞迴刪除所有關聯實體檔。
- 刪除失敗時顯示未完成項目，不回報成功。
- App 進入 inactive／background 前顯示隱私遮罩。
- 備份與還原改為可取消、可回滾的流程。

#### Gate

- 原始內容沒有任何可直接編輯入口。
- 建立、修改、備份、還原、刪除後的檔案數量與資料庫一致。
- 錯誤密碼、損壞備份及還原中斷不造成半套資料。
- App 切換器截圖不顯示事件內容。

### M1：聊天截圖整理

#### 交付內容

- Share Extension 保留使用者選取順序。
- 一次匯入多張截圖並歸為同一 `ConversationGroup`。
- 手動拖曳排序與重複區段標記。
- OCR 保留文字與畫面座標。
- 訊息分段草稿：左側、右側、系統訊息、引用、附件、無法辨識。
- 使用者確認哪一側是本人，其他人物以中性名稱建立。
- 分別保存截圖建立時間、匯入時間、畫面顯示時間與確認日期。
- 可將一組對話關聯至新事件或既有事件。

#### Gate

- 5～10 張連續聊天截圖能完成排序、人物確認與事件關聯。
- 未確認的 OCR／人物／日期不會寫入正式事件欄位。
- 裁切、標註、遮罩及不確定狀態能被保存與匯出。
- 「已收回訊息」只保留畫面可見事實。

### M2：專業案件包

#### 交付內容

- 多事件案件選擇器。
- 一頁案件總覽。
- 事件時間線。
- 事件詳情與原始／修正版區分。
- 聊天截圖縮圖、附件編號、OCR 確認稿與人物映射。
- PDF 產製聲明與不確定事項。
- 完整 ZIP：PDF、使用者選定的原始附件及 manifest。
- manifest 保存檔名、SHA-256、匯入時間、來源說明與關聯事件。
- 匯出前敏感資料確認。

#### Gate

- 3～5 名勞動法律師、工會或勞權工作者可在五分鐘內理解案件脈絡。
- PDF 與 ZIP 內附件編號一致。
- 匯出前後重新驗證 SHA-256。
- 大量附件能正確分頁、產製、取消及重試。

### M3：TestFlight 外部測試準備

#### 交付內容

- Apple Developer Team 與自動簽署。
- 註冊並啟用 `group.tw.dayi.trace`。
- 主 App 與 Share Extension 真機測試。
- Onboarding：產品用途、不是法律判定、資料位置與備份風險。
- 隱私權政策、測試條款與聯絡方式。
- App 內回饋入口，不夾帶事件內容。
- 測試版診斷匯出，由使用者主動確認後產生。
- TestFlight 版本號、Release Notes 與已知問題。
- Threads 招募貼文及測試者篩選表。

#### Gate

- 至少兩台不同世代 iPhone 完成安裝、匯入、OCR、PDF、備份與還原。
- App Group／Share Extension 在正式簽署環境可用。
- 無重大資料遺失、誤匯出或刪除不完整問題。
- 測試者可清楚說明資料是否離開裝置。

### M4：語音轉錄與 Apple Intelligence

M4 是第一輪 External Beta Gate，完成後才能對外招募 Threads 測試者。

#### 交付內容

- App 內可見式錄音：開始、暫停、繼續、停止、時間與檔案狀態。
- 從 Files 匯入既有音檔並保存原始檔、metadata 與 SHA-256。
- iOS 26 以上以 SpeechAnalyzer／SpeechTranscriber 進行裝置端逐字稿。
- 逐字稿分段、時間範圍、原始辨識結果與使用者確認版本。
- Foundation Models guided generation，產生人物、時間、具體行為、工作影響、資訊缺口與最多三個中性補問。
- AI 草稿必須逐欄確認後才能建立事件修訂版。
- 顯示 Apple Intelligence 的可用、裝置不符、模型未就緒、語言不支援及關閉狀態。
- 舊裝置或模型不可用時提供固定問答，不上傳第三方模型。
- Prompt、輸出結構與模型版本記錄，以便 OS 更新後回歸測試。

#### Gate

- 至少一台支援裝置完成「錄音 → 逐字稿 → 確認 → AI 草稿 → 事件卡 → PDF」。
- 至少一台不支援 Apple Intelligence 的裝置能完成錄音與手動整理，且不出現阻斷。
- 中斷、來電、耳機切換、低儲存空間及 App 終止不會產生無法管理的孤兒音檔。
- AI 不得把未出現在來源中的動機、人物、時間或法律結論寫成事實。
- 使用者拒絕 AI 草稿後，原始逐字稿、附件與事件不受修改。

### M5：法規資訊與商業驗證

M5 不阻擋第一輪 External Beta：

- StoreKit 案件包價格測試：NT$290／490／690。
- 法規資訊對照與來源版本管理。
- 功能旗標與測試群組。

## 6. 工作流與相依關係

| 工作流 | 前置條件 | 主要產出 |
|---|---|---|
| 資料完整性 | 現有 SwiftData schema | 不可變原始資料、版本、完整刪除 |
| 對話整理 | 資料完整性 | 多圖排序、人物、訊息與事件關聯 |
| 案件包 | 對話整理、專業人士輸出回饋 | PDF、ZIP、manifest |
| 語音轉錄 | 資料完整性、音檔儲存 | 原始音檔、逐字稿及確認版本 |
| Apple Intelligence | 已確認 OCR／逐字稿 | 結構化草稿、缺口與中性補問 |
| 公開測試 | M0～M4 Gate 通過 | TestFlight、隱私文件、回饋流程 |
| 付費驗證 | 案件包價值成立 | StoreKit 購買實驗 |

## 7. 測試策略

### 自動化

- Model：事件、附件、修訂與關聯 CRUD。
- File Store：原子寫入、雜湊、重新驗證與完整刪除。
- Share Import：順序、批次、重試與損壞 manifest。
- OCR：繁體中文、英文混合、空結果及錯誤圖片。
- Audio：錄音狀態、音訊中斷、路由切換、檔案完成與 SHA-256。
- Speech：支援語系、資產未下載、空音訊、長音檔、取消與逐字稿版本。
- Foundation Models：guided generation、availability、語言、guardrail、context window 及取消。
- Prompt Evaluation：固定匿名測試集檢查遺漏、臆測、語氣與結構正確性。
- Backup：建立→刪除本機資料→還原→逐檔比對雜湊。
- Export：PDF 頁數、附件索引、ZIP manifest 與原始檔一致。
- UI：建立事件、加入截圖、確認 OCR、匯出案件包。

### 真機

- Face ID 成功、取消、鎖定與裝置密碼降級。
- Photos／Files／Share Extension。
- App Group 正式簽署。
- App 切換器遮罩。
- 低儲存空間、離線、背景中斷與大附件。
- `zh-TW` OCR 的真實 LINE 截圖樣本。
- 繁體中文語音的長短音檔、多人重疊與背景噪音。
- 支援與不支援 Apple Intelligence 的裝置降級體驗。

## 8. 公開測試指標

### 核心漏斗

1. 安裝並完成用途與隱私說明。
2. 建立第一件事件。
3. 加入至少一張截圖。
4. 完成 OCR／人物／日期確認。
5. 關聯事件。
6. 錄音或匯入音檔並確認逐字稿。
7. 接受、修改或拒絕 AI 事件草稿。
8. 匯出 PDF 或完整案件包。

### 品質與信任

- 原始附件保存成功率。
- OCR 確認完成率與平均修正率。
- 多圖排序完成率。
- 語音轉錄完成率、失敗率與逐字稿修正比例。
- AI 草稿逐欄接受率、刪改率與臆測回報率。
- 支援及不支援 Apple Intelligence 裝置的流程完成率差異。
- 備份及還原成功率。
- 完整刪除測試通過率。
- 敏感資料誤匯出事件數。
- 專業人士理解事件所需時間。

不以 DAU、簽到天數或一般社群留存作為首輪成功標準。

## 9. Threads 招募方案

### 第一輪：封閉測試

- 5～10 位測試者。
- 使用模擬、去識別或測試者自行匿名化的真實內容。
- 先驗證資料安全、理解成本與流程摩擦。

### 第二輪：Threads 公開招募

- 招募 20～30 位，目標至少 10 位完成核心漏斗。
- 貼文不使用「AI 判斷霸凌」或「保證蒐證有效」等主張。
- 主張聚焦：「把零散工作對話與事件整理成可確認的事件包」。
- 公開已知限制、支援裝置與資料保存方式。

## 10. 主要風險

| 風險 | 影響 | 處理方式 |
|---|---|---|
| 使用者誤認 App 可判定霸凌 | 法律與信任風險 | 全流程使用中性語言與產製聲明 |
| 原始資料被覆寫或遺失 | 產品核心失敗 | 不可變資料層、原子操作、雜湊與測試 |
| 第三人個資進入案件包 | 隱私風險 | 匯出前確認、遮罩建議、使用者控制 |
| App Group／簽署不正確 | Share Extension 無法使用 | 真機 Gate 與兩 target 同組設定 |
| 備份密碼遺失 | 無法還原 | 明確警告、密碼不保存、不提供假復原承諾 |
| OCR 誤判改變語意 | 案件內容失真 | 原始 OCR、修正版、使用者確認三層分離 |
| 功能過多延誤驗證 | 無法及時取得市場訊號 | M0～M4 僅保留完整核心旅程，商業功能延後 |
| Apple Intelligence 裝置覆蓋不足 | 測試樣本偏少 | 招募時分層，至少涵蓋支援及不支援裝置 |
| 語音或 AI 產生錯誤內容 | 使用者誤信且影響案件整理 | 保存原始版本、強制確認、顯示來源及禁止法律結論 |
| OS 更新改變模型輸出 | Prompt 品質退化 | 保存 prompt/model version 並以固定資料集回歸測試 |

## 11. 公開測試 Definition of Done

只有以下條件全部成立，版本才可標記為 External Beta Candidate：

- 原始資料不可覆寫且可追溯修訂。
- 刪除事件會移除所有關聯檔案與衍生資料。
- 多張聊天截圖可排序、確認人物並關聯事件。
- 可見式錄音及既有音檔匯入能保存完整原始檔。
- 支援裝置可完成 SpeechAnalyzer 逐字稿與使用者確認。
- Apple Intelligence 可產生結構化事件草稿並保留來源追溯。
- 不支援 Apple Intelligence 的裝置可順利使用固定問答降級流程。
- 多事件 PDF／ZIP 案件包可生成並通過雜湊核對。
- App 切換器不顯示敏感內容。
- 備份還原往返測試通過。
- 主 App 與 Share Extension 真機簽署測試通過。
- 隱私權政策、測試條款、聯絡與回饋入口完成。
- CI 編譯、自動化測試及關鍵 UI 測試通過。
- 已知問題不包含資料遺失、誤刪、誤匯出或未授權分享。
