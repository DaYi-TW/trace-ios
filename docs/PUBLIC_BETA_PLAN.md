# 留痕 Trace：公開測試版產品與開發規劃書

> 文件版本：1.0
>
> 文件日期：2026-07-15
>
> 產品階段：可編譯探索版 → TestFlight 外部測試
>
> 初期平台：iPhone／iOS 17 以上
>
> 招募通路：Threads、專業人士與勞權社群轉介

## 1. 規劃結論

目前 Repo 已具有可編譯的 SwiftUI App、SwiftData、本機附件保存、OCR、Share Extension、Face ID、加密備份、單事件 PDF、視覺設計與 App Icon。

下一個目標不是增加更多展示功能，而是完成一個能安全交給真實使用者的外部測試版：

> 使用者能主動匯入一組聊天截圖，保留不可覆寫的原始檔，確認 OCR、人物、順序與事件關聯，最後輸出可交給專業人士理解的案件包；整個過程預設不離開裝置。

TestFlight 公開前必須先補齊資料不可變、完整刪除、對話整理、案件包、真機安全測試與隱私文件。SpeechAnalyzer、Foundation Models、法規 AI 與付費功能不列為首輪公開測試阻擋條件。

## 2. 目前基準

### 已完成

- SwiftUI App 與 SwiftData 資料模型。
- 建立、編輯與刪除事件的基本畫面。
- 從 Photos、Files 匯入圖片、PDF 與音檔。
- Share Extension 接收多張圖片並透過 App Group 暫存。
- 原始附件保存、匯入時間與 SHA-256 計算。
- Vision 本機 OCR 與使用者確認欄位。
- 單事件摘要 PDF。
- Face ID／Touch ID／裝置密碼畫面保護。
- 密碼加密備份及還原的第一版。
- 「溫和留痕／案件編目／私密保險箱」視覺系統。
- 1024×1024 App Icon。
- GitHub Actions macOS Simulator 編譯驗證。

### 尚未達到公開測試標準

- 原始陳述與原始 OCR 仍可從 UI 直接修改。
- 缺少資料修訂版本與稽核紀錄。
- 刪除事件時，附件實體檔可能留在裝置中。
- Share Extension 的多圖完成順序可能與使用者選取順序不同。
- 沒有聊天截圖專用的人物、訊息順序及跨圖分組流程。
- PDF 仍是單事件摘要，不是多事件案件包。
- 缺少完整原始附件 ZIP 與 manifest。
- App 切換器隱私遮罩尚未完成。
- 備份、刪除、OCR 與匯出的自動化測試尚未建立。
- 尚未完成 Apple Developer App Group、簽署與真機驗證。
- 尚無隱私權政策、測試者同意與 App 內回饋入口。

## 3. 公開測試目標

### 使用者目標

使用者可在不理解法律術語的情況下完成：

```text
保存原始截圖
→ 確認對話文字、人物、日期與順序
→ 關聯一件或多件工作事件
→ 預覽事件時間線
→ 匯出 PDF／完整附件包
```

### 產品驗證目標

首輪 TestFlight 只驗證以下問題：

1. 使用者是否願意把真實但可匿名化的聊天截圖加入留痕。
2. OCR 確認、人物對應與事件關聯是否比 Word／相簿整理省力。
3. 專業人士是否能更快理解事件脈絡與缺漏。
4. 使用者是否信任資料預設保留在裝置上。
5. 完整案件包是否具有可付費價值。

## 4. 不可妥協原則

1. 原始附件、原始陳述與系統原始 OCR 不可覆寫。
2. 所有修正以新版本保存，並能追溯來源。
3. SHA-256 只驗證匯入後完整性，不宣稱證明真偽。
4. AI 只產生待確認草稿，不作法律結論。
5. App 不讀取 LINE 帳號、聊天資料庫或通知內容。
6. 不提供隱蔽錄音或背景持續錄音。
7. 使用者控制備份、匯出、分享與刪除。
8. 不因加入 AI 而讓舊裝置無法完成核心流程。

## 5. 開發階段與 Gate

本規劃採 Gate 制，不以固定週數自動前進。每階段通過驗收才進入下一階段。

### M0：資料完整性與安全刪除

#### 交付內容

- 建立 `EventRevision`，原始陳述唯讀，修正另存版本。
- 建立 `OCRResult` 與 `ConfirmedTranscript` 分層。
- 新增附件完整性重新驗證。
- 刪除單一附件時移除資料庫紀錄、原始檔及衍生檔。
- 刪除事件時遞迴刪除所有關聯實體檔。
- 刪除失敗時顯示未完成項目，不回報成功。
- App 進入 inactive／background 前顯示隱私遮罩。
- 備份與還原改為可取消、可回滾的流程。

#### Gate

- 原始內容沒有任何可直接編輯入口。
- 建立、修改、備份、還原、刪除後的檔案數量與資料庫一致。
- 錯誤密碼、損壞備份及還原中斷不造成半套資料。
- App 切換器截圖不顯示事件內容。

### M1：聊天截圖整理

#### 交付內容

- Share Extension 保留使用者選取順序。
- 一次匯入多張截圖並歸為同一 `ConversationGroup`。
- 手動拖曳排序與重複區段標記。
- OCR 保留文字與畫面座標。
- 訊息分段草稿：左側、右側、系統訊息、引用、附件、無法辨識。
- 使用者確認哪一側是本人，其他人物以中性名稱建立。
- 分別保存截圖建立時間、匯入時間、畫面顯示時間與確認日期。
- 可將一組對話關聯至新事件或既有事件。

#### Gate

- 5～10 張連續聊天截圖能完成排序、人物確認與事件關聯。
- 未確認的 OCR／人物／日期不會寫入正式事件欄位。
- 裁切、標註、遮罩及不確定狀態能被保存與匯出。
- 「已收回訊息」只保留畫面可見事實。

### M2：專業案件包

#### 交付內容

- 多事件案件選擇器。
- 一頁案件總覽。
- 事件時間線。
- 事件詳情與原始／修正版區分。
- 聊天截圖縮圖、附件編號、OCR 確認稿與人物映射。
- PDF 產製聲明與不確定事項。
- 完整 ZIP：PDF、使用者選定的原始附件及 manifest。
- manifest 保存檔名、SHA-256、匯入時間、來源說明與關聯事件。
- 匯出前敏感資料確認。

#### Gate

- 3～5 名勞動法律師、工會或勞權工作者可在五分鐘內理解案件脈絡。
- PDF 與 ZIP 內附件編號一致。
- 匯出前後重新驗證 SHA-256。
- 大量附件能正確分頁、產製、取消及重試。

### M3：TestFlight 外部測試準備

#### 交付內容

- Apple Developer Team 與自動簽署。
- 註冊並啟用 `group.tw.dayi.trace`。
- 主 App 與 Share Extension 真機測試。
- Onboarding：產品用途、不是法律判定、資料位置與備份風險。
- 隱私權政策、測試條款與聯絡方式。
- App 內回饋入口，不夾帶事件內容。
- 測試版診斷匯出，由使用者主動確認後產生。
- TestFlight 版本號、Release Notes 與已知問題。
- Threads 招募貼文及測試者篩選表。

#### Gate

- 至少兩台不同世代 iPhone 完成安裝、匯入、OCR、PDF、備份與還原。
- App Group／Share Extension 在正式簽署環境可用。
- 無重大資料遺失、誤匯出或刪除不完整問題。
- 測試者可清楚說明資料是否離開裝置。

### M4：裝置端智慧整理與商業驗證

M4 不阻擋第一輪 TestFlight，可在 M0～M3 穩定後分批開啟：

- SpeechAnalyzer／SpeechTranscriber 事後口述與既有音檔逐字稿。
- Foundation Models 結構化事件與最多三個中性補問。
- 固定問答與不支援裝置降級流程。
- StoreKit 案件包價格測試：NT$290／490／690。
- 法規資訊對照與來源版本管理。
- 功能旗標與測試群組。

## 6. 工作流與相依關係

| 工作流 | 前置條件 | 主要產出 |
|---|---|---|
| 資料完整性 | 現有 SwiftData schema | 不可變原始資料、版本、完整刪除 |
| 對話整理 | 資料完整性 | 多圖排序、人物、訊息與事件關聯 |
| 案件包 | 對話整理、專業人士輸出回饋 | PDF、ZIP、manifest |
| 公開測試 | 前三項 Gate 通過 | TestFlight、隱私文件、回饋流程 |
| 語音與 AI | 核心流程穩定 | 可選裝置端效率層 |
| 付費驗證 | 案件包價值成立 | StoreKit 購買實驗 |

## 7. 測試策略

### 自動化

- Model：事件、附件、修訂與關聯 CRUD。
- File Store：原子寫入、雜湊、重新驗證與完整刪除。
- Share Import：順序、批次、重試與損壞 manifest。
- OCR：繁體中文、英文混合、空結果及錯誤圖片。
- Backup：建立→刪除本機資料→還原→逐檔比對雜湊。
- Export：PDF 頁數、附件索引、ZIP manifest 與原始檔一致。
- UI：建立事件、加入截圖、確認 OCR、匯出案件包。

### 真機

- Face ID 成功、取消、鎖定與裝置密碼降級。
- Photos／Files／Share Extension。
- App Group 正式簽署。
- App 切換器遮罩。
- 低儲存空間、離線、背景中斷與大附件。
- `zh-TW` OCR 的真實 LINE 截圖樣本。

## 8. 公開測試指標

### 核心漏斗

1. 安裝並完成用途與隱私說明。
2. 建立第一件事件。
3. 加入至少一張截圖。
4. 完成 OCR／人物／日期確認。
5. 關聯事件。
6. 匯出 PDF 或完整案件包。

### 品質與信任

- 原始附件保存成功率。
- OCR 確認完成率與平均修正率。
- 多圖排序完成率。
- 備份及還原成功率。
- 完整刪除測試通過率。
- 敏感資料誤匯出事件數。
- 專業人士理解事件所需時間。

不以 DAU、簽到天數或一般社群留存作為首輪成功標準。

## 9. Threads 招募方案

### 第一輪：封閉測試

- 5～10 位測試者。
- 使用模擬、去識別或測試者自行匿名化的真實內容。
- 先驗證資料安全、理解成本與流程摩擦。

### 第二輪：Threads 公開招募

- 招募 20～30 位，目標至少 10 位完成核心漏斗。
- 貼文不使用「AI 判斷霸凌」或「保證蒐證有效」等主張。
- 主張聚焦：「把零散工作對話與事件整理成可確認的事件包」。
- 公開已知限制、支援裝置與資料保存方式。

## 10. 主要風險

| 風險 | 影響 | 處理方式 |
|---|---|---|
| 使用者誤認 App 可判定霸凌 | 法律與信任風險 | 全流程使用中性語言與產製聲明 |
| 原始資料被覆寫或遺失 | 產品核心失敗 | 不可變資料層、原子操作、雜湊與測試 |
| 第三人個資進入案件包 | 隱私風險 | 匯出前確認、遮罩建議、使用者控制 |
| App Group／簽署不正確 | Share Extension 無法使用 | 真機 Gate 與兩 target 同組設定 |
| 備份密碼遺失 | 無法還原 | 明確警告、密碼不保存、不提供假復原承諾 |
| OCR 誤判改變語意 | 案件內容失真 | 原始 OCR、修正版、使用者確認三層分離 |
| 功能過多延誤驗證 | 無法及時取得市場訊號 | M0～M3 為唯一公開測試必做範圍 |

## 11. 公開測試 Definition of Done

只有以下條件全部成立，版本才可標記為 External Beta Candidate：

- 原始資料不可覆寫且可追溯修訂。
- 刪除事件會移除所有關聯檔案與衍生資料。
- 多張聊天截圖可排序、確認人物並關聯事件。
- 多事件 PDF／ZIP 案件包可生成並通過雜湊核對。
- App 切換器不顯示敏感內容。
- 備份還原往返測試通過。
- 主 App 與 Share Extension 真機簽署測試通過。
- 隱私權政策、測試條款、聯絡與回饋入口完成。
- CI 編譯、自動化測試及關鍵 UI 測試通過。
- 已知問題不包含資料遺失、誤刪、誤匯出或未授權分享。
